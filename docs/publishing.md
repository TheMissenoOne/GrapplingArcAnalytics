# Publishing the site — cheapest reliable hosting (BRL)

The public site (`../GrapplingArc`) is **fully static**: data is generated by
`export/site_data.py` and committed, the `site/` bundle is self-contained, and the root
`index.html` redirects into it. Static + committed → hosting cost can be **R$0**; the only real
recurring cost is a domain.

## Recommendation — Cloudflare Pages + `.com.br` ≈ **R$40/year**

| Item | Provider | Cost (BRL) |
|---|---|---|
| Hosting (CDN, SSL, bandwidth) | Cloudflare Pages (free plan) | **R$0** |
| Domain `grapplingarc.com.br` | Registro.br | **R$40/yr** (no renewal markup) |
| **Total** | | **≈ R$40/yr** |

Why Cloudflare Pages: free plan has **unlimited bandwidth/requests**, a **300+ PoP** edge CDN
(fast in Brazil), free auto-SSL, custom domains, and Git-connected auto-deploys — strictly
better reach/perf than GitHub Pages at the same R$0. Why Registro.br: cheapest `.com.br`
(R$40/yr, no first-year-bait renewal hikes that other registrars charge), multi-year discounts.

## Steps

1. **Domain** — register `grapplingarc.com.br` at https://registro.br (R$40/yr; pay multiple
   years for the progressive discount).
2. **Cloudflare** — create a free account; add the domain (move its nameservers to Cloudflare,
   or keep Registro.br DNS and add a CNAME to the Pages project).
3. **Pages** — Cloudflare dashboard → Pages → **Connect to Git** → the `GrapplingArc` repo.
   Build config: either
   - **Jekyll**: build command `jekyll build`, output dir `_site` (matches the current Actions
     build), or
   - **None / static**: serve the repo root directly — the self-contained `site/` bundle + root
     redirect already work without a build.
   Then **Custom domains → add `grapplingarc.com.br`** (SSL provisions automatically).
4. **Deploy** stays the same workflow: push to `main` → Cloudflare auto-builds & deploys.
   Keep GitHub Pages as a fallback/staging origin, or disable it once Cloudflare is live.

## Zero-cost alternative (defer the domain)
Keep GitHub Pages (already wired) **or** Cloudflare Pages on the free `*.pages.dev` /
`*.github.io` subdomain — **R$0/yr**, no domain. Add the `.com.br` later when branding matters.

## Notes
- No backend, no database calls from the site → no server/egress costs ever; cost is flat
  regardless of traffic.
- The generated data (`*-data.js`, `breakdown-*`, `event-*`, `grapple-*`) is committed, so the
  host only ever serves static files.
